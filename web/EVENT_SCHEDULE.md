# 祭イベントスケジュールの登録方法

マップは Google スプレッドシートの **`event_schedule`** シート（名前は変更可）を読み、**開始時刻の N 分前**から**終了時刻まで**の間だけ、その場所にピンを表示します。

## 前提

- 店舗マスタなどと**同じスプレッドシート**を使い、ブラウザから **gviz（Visualization API）で読める公開設定**にしておいてください（「リンクを知っている全員が閲覧可」など）。
- フロントの `secrets.local.js`（または GitHub Actions の生成ファイル）に **`SHEET_ID`** が正しく入っていること。
- シート名を変える場合は **`EVENTS_SHEET`** を設定します（未設定時は `event_schedule`）。  
  例: `secrets.example.js` の `window.__SHIMODA_MAP_SECRETS__.EVENTS_SHEET`

## シートを用意する方法

1. **手動**  
   同じブック内にシート `event_schedule` を追加し、下記の**1行目をヘッダ**として入力します。

2. **Google Apps Script（任意）**  
   `gas-line-webhook.js` の **`setupSheets()`** を実行すると、`event_schedule` が無いときだけヘッダ付きのシートが末尾に作成されます（Webhook 用 GAS をプロジェクトに反映したうえで、エディタから1回実行）。

## 列の意味（1行目ヘッダ）

| 列 | ヘッダ例 | 必須 | 説明 |
|----|-----------|------|------|
| A | `event_id` | 任意 | 識別子（空なら行番号ベースの内部 ID）。同一ブック内で分かりやすくするため推奨。 |
| B | `title` | **必須** | イベント名（日本語）。Popup のタイトルに使います。 |
| C | `start_at` | **必須** | **開始日時**。セルの日付書式、または `parseSheetDate` が解釈できる文字列（後述）。 |
| D | `lat` | **必須** | 緯度（数値）。 |
| E | `lng` | **必須** | 経度（数値）。 |
| F | `emoji` | 任意 | ピンに表示する絵文字。空なら `🎪`。 |
| G | `duration_minutes` | 任意 | 開催時間（分）。**終了時刻 = 開始 + この分数**。空なら **60**。 |
| H | `lead_minutes` | 任意 | **何分前からピンを出すか**。空ならアプリ既定（`EVENT_SCHEDULE_LEAD_MINUTES`、通常 **30**）。 |
| I | `hidden` | 任意 | **`TRUE`**（またはブールの true）/`1`/`"TRUE"` のとき、その行は**読み込まれません**。通常は**空欄**のまま。チェックボックス列にした場合は「**チェックあり＝非掲載**」です（未チェック＝`false` は掲載対象）。 |
| J | `title_en` | 任意 | 英語タイトル。言語が英語のとき Popup に使います。空なら `title` と同じ扱い。 |
| K | `image_url` | 任意 | **イベント画像 URL**（`https://...`）。**ポップアップとピン（円形クロップ）の両方**に表示。空ならピン・ポップアップとも **emoji のみ**。読み込み失敗時はピンは emoji にフォールバック。 |

※ **データは2行目から**入れます（1行目はヘッダ）。  
※ 既存ブックに列が J までしかない場合は、**K 列にヘッダ `image_url` を追加**してください。

## 登録例

| event_id | title | start_at | lat | lng | emoji | duration_minutes | lead_minutes | hidden | title_en | image_url |
|----------|-------|----------|-----|-----|-------|------------------|--------------|--------|----------|-----------|
| ev_001 | 大道芸 | 2026-07-19 12:00:00 | 34.6801 | 138.9430 | 🎪 | 60 | 30 | | Street show | https://example.com/flyer.jpg |

- この例では **11:30** からピンが表示され、**13:00**（開始から60分後）まで表示されます。

## 日時の入れ方とタイムゾーン

- **推奨**: スプレッドシートの**タイムゾーンを祭り当日の想定（日本なら JST）に合わせる**うえで、日時セルを「日付」と「時刻」として入力する。
- **ISO 8601**（例: `2026-07-19T12:00:00+09:00`）のようにオフセット付きで書くと、解釈がブレにくくなります。
- gviz 経由で配列として渡る形式は、ブラウザ側ではローカル日時として組み立てられるため、**シートのタイムゾーン・入力形式・閲覧者の環境**がずれると表示タイミングがずれることがあります。運用前に実際のピン出現時刻を確認してください。

## マップ上の挙動

- **表示ウィンドウ**: `現在時刻 >= 開始時刻 - lead_minutes` かつ `現在時刻 < 終了時刻` のときだけピンを描画します。
- **ポップアップ**: マーカーをタップすると Mapbox のポップアップが開き、タイトル・時間帯・状態に加え、**K 列 `image_url` があるときだけ**その画像を最大高さ 200px で 1 枚表示します（読み込み失敗時は非表示）。**ピン上**でも同じ URL を円形マーカー内に表示し、失敗時は **emoji** に戻ります。
- **レイヤー**: マップのレイヤーパネル **「祭スケジュール」** をオフにするとスケジュールのピンだけ非表示にできます（店舗ピンとは独立）。
- **更新**: シートの再取得は約 **60秒ごと**（`EVENT_SCHEDULE_POLL_INTERVAL`）。表示の切り替わりは約 **1分ごと**に再評価されます（`EVENT_SCHEDULE_TICK_MS`）。

## コード側の既定値（参考）

`web/index.html` の `CONFIG` より:

- `EVENT_SCHEDULE_LEAD_MINUTES`: 30（列 H が空のときの「何分前から」）
- `EVENT_SCHEDULE_POLL_INTERVAL`: 60000（ms）
- `EVENT_SCHEDULE_TICK_MS`: 60000（ms）

変更する場合は `index.html` の `CONFIG` を編集してください（ secrets ではなくコード側の定数です）。

## トラブルshooting

| 症状 | 確認すること |
|------|----------------|
| ピンが一切出ない | シート名が `EVENTS_SHEET` / 既定 `event_schedule` と一致しているか。公開リンクで gviz が読めるか。 |
| 時間になっても出ない | `title`・`start_at`・`lat`・`lng` が空や不正でないか。`hidden` が誤って **`TRUE` になっていないか**（表示したい行は空欄または未チェック）。**現在時刻が「開始 − lead」～「終了」の外**になっていないか。 |
| 英語でタイトルがおかしい | `title_en` を埋める。 |
| 画像が出ない | K 列 URL が `https://` で公開読み取りできるか。CORS／Referer でブロックされていないか。別ドメインなら自サイトや Imgur・Drive 直リンク等で試す。 |
